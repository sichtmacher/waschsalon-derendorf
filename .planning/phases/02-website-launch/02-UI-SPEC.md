# Phase 2 — UI Design Contract

> Visual and interaction contract for the Waschsalon Derendorf Website Launch. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind v4 `@theme` tokens) |
| Preset | not applicable |
| Component library | none — Astro components + inline React islands |
| Icon library | Inline SVG (24x24 viewBox, `stroke-width: 2`, `stroke-linecap: round`, `stroke-linejoin: round`) — matches existing Phase 1 Lucide-style icons in `index.astro` / `GoogleMap.astro` |
| Font | Body: **Nunito** (via `--font-body`, Astro Fonts API, self-hosted) · Heading: **Baloo 2** (via `--font-heading`, self-hosted) |

**Rationale for no component library:** Project is a single-page marketing site with 5 sections (Hero, Öffnungszeiten, Preise, FAQ, Anfahrt) plus two legal sub-pages. Tailwind v4 `@theme` tokens from Phase 1 already cover all primitives. Adding shadcn/Radix would pull React-island overhead onto pages that are currently 90% static Astro. Decision: continue with manual Astro components (`StickyNav.astro`, `PriceCard.astro`, `FaqAccordion.astro`) and reuse Phase 1's `BaseLayout.astro`, `GoogleMap.astro`, `WarpBackground.tsx`.

**Existing token source:** `src/styles/global.css` Tailwind v4 `@theme` block.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside pills, inline badge padding |
| sm | 8px | Compact element spacing, accordion chevron gap |
| md | 16px | Default element spacing, card internal padding, gap between FAQ items |
| lg | 24px | Card padding, section padding on mobile, gap between Pricing Cards |
| xl | 32px | Section padding desktop, gap between `<section>` stacks on mobile |
| 2xl | 48px | Vertical spacing between major sections on tablet |
| 3xl | 64px | Vertical spacing between major sections on desktop, hero top/bottom breathing room |

Section vertical rhythm: mobile `py-12` (48px), desktop `py-20` (80px → uses Tailwind stock, acceptable multiple of 4).

**Exceptions:**
- **Touch targets**: Route-planen CTA and Accordion headers enforce `min-height: 44px` (WCAG 2.5.5 AAA) — satisfied via `py-2.5` + `px-5` on buttons (already used in `GoogleMap.astro` pattern).
- **Hero glass pills** use `px-4 py-2.5` and `px-5 py-2.5` (20px / 10px) inherited from Phase 1 — locked for visual continuity with coming-soon aesthetic, no change.

## Typography

Exactly 4 sizes, exactly 2 weights. Heading face is **Baloo 2**, body face is **Nunito** (both already loaded via Astro Fonts API in `BaseLayout.astro`).

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 (regular) | 1.5 | Nunito (`--font-body`) |
| Label | 14px | 600 (semibold) | 1.4 | Nunito — used for pill-labels, pricing metadata, sticky-nav links, FAQ category headers |
| Heading | 24px (mobile) / 32px (desktop) | 600 (semibold) | 1.2 | Baloo 2 (`--font-heading`) — section titles (Öffnungszeiten, Preise, FAQ, Anfahrt) and Pricing-Card titles |
| Display | 40px (mobile) / 56px (desktop) | 600 (semibold) | 1.1 | Baloo 2 — hero H1 "Waschsalon Derendorf" (already present in Phase 1) |

**Rules:**
- Only these 2 weights are used. No 300, 500, 700, or 800.
- Hero subtitle "Dein neuer SB-Waschsalon in Derendorf" uses Body size at 18–20px (Tailwind `text-lg`/`text-xl`) — treated as an *enlarged Body*, not a 5th scale value, consistent with Phase 1.
- Price values ("ab 5,00 €") use Heading scale (24/32px) for visual weight in Pricing Cards.
- FAQ question text (accordion trigger) uses Body size at 600 weight.
- FAQ answer body uses Body size at 400 weight.
- German: no Latin ligatures, no small caps, `lang="de"` at `<html>` root (Phase 1 already sets this).

## Color

Palette tokens live in `src/styles/global.css` under Tailwind v4 `@theme`. Reuse — do not re-declare.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFFFE` (body bg) + `#FFFFFF` (surfaces) | Page background below hero, Pricing Card surfaces, FAQ accordion background, Anfahrt section background, Impressum/Datenschutz full surface |
| Secondary (30%) | `#F0FDFA` (`--color-surface`) + `#E0F7FA` (`--color-brand-light`) | Section alternation stripes (Öffnungszeiten + FAQ sit on `--color-surface`), map placeholder bg, accordion hover state, small info callouts |

| Accent Hover/Dark | `#0E7490` (`--color-brand-dark`) | Primary CTA hover/active state, link hover |
| Secondary Accent | `#F59E0B` (`--color-accent`, Amber) | **Reserved for: "Eröffnung 2026" badge background border, exactly one highlight per Pricing Card (the feature pill denoting the biggest machine "15 kg"), FAQ section lead-paragraph emphasis word** — never used on interactive elements (keeps Teal as the sole CTA color) |
| Destructive | not used | Phase 2 has zero destructive actions (no delete, no cancel, no logout) |
| Text primary | `#1E293B` (`--color-text`) | All body copy, headings, Impressum/Datenschutz body |
| Text muted | `#64748B` (`--color-text-muted`) | FAQ secondary metadata, footer links, pricing-card "Dauer" labels, pre-opening sub-text |
| Hero overlay | Glass: `rgba(255,255,255,0.12)` surface, `rgba(255,255,255,0.20)` pill, `rgba(255,255,255,0.30)` border, `backdrop-filter: blur(8–10px)` | Inherited from Phase 1, used exclusively inside Hero section over the Warp shader |

**60/30/10 allocation rationale:**
- Below-the-fold content is a light airy marketing site — white surfaces dominate.
- Teal appears on ~10% of rendered pixels: CTAs, icon chips, status pills, link text — enough to feel branded, not garish.
- Amber is a guardrail accent — used sparingly to flag the Eröffnung badge and a single feature-emphasis per card. Never for CTAs, never for hover states.

**Contrast verification (WCAG AA required):**
- `#1E293B` on `#FFFFFF` = 14.8:1 ✅
- `#1E293B` on `#F0FDFA` = 14.4:1 ✅
- `#FFFFFF` on `#0891B2` = 4.6:1 ✅ (CTA button)
- `#0891B2` on `#FFFFFF` = 4.6:1 ✅ (link text)
- `#64748B` on `#FFFFFF` = 4.6:1 ✅ (muted text)

## Accessibility Contract

| Requirement | Mechanism |
|-------------|-----------|
| Color contrast AA | Verified in Color section above; all pairs ≥ 4.5:1 for normal text, ≥ 3:1 for large |
| Keyboard navigation | Tab order: Skip-to-content link → Hero CTAs → Sticky Nav (once visible) → Pricing Cards (no focus — purely visual) → FAQ triggers → Anfahrt links → Map button → Footer links |
| Screen reader | All icons decorative get `aria-hidden="true"`; tel/mail/map links get `aria-label` with address context |
| Reduced motion | `prefers-reduced-motion: reduce` disables shader speed, card lift, accordion animation |
| FAQ semantic | `<details>` + `<summary>` native element (simpler, accessible by default) OR `<button aria-expanded>` + `<div role="region">` — planner may choose, both pass |
| Landmark regions | `<header>` (sticky nav), `<main>`, `<footer>`, `<section aria-labelledby="preise-heading">` etc. |
| Skip link | "Zum Hauptinhalt springen" — visually hidden until focus, then teal CTA style |
| Language | `<html lang="de">` (already set in Phase 1 BaseLayout) |
